# Loki Mode: Autonomous Task Orchestration

## Overview
Loki Mode is an advanced multi-agent orchestration skill designed exclusively for the Antigravity IDE. It acts as the primary execution engine for translating complex Product Requirements Documents (PRDs) into production-ready software. By managing end-to-end task completion, Loki Mode transforms high-level architectural visions into a synchronized digital assembly line.

## Core Capabilities
* **End-to-End Execution:** Autonomously reads vague PRDs, generates concrete, step-by-step Implementation Plans, requests human sign-off, and executes the blueprint.
* **Parallel Assembly Line:** Orchestrates isolated Git worktrees, allowing parallel AI sessions to simultaneously handle feature development, unit testing, and documentation generation. 
* **Automated Integration:** Autonomously auto-merges completed components from parallel worktrees back into a cohesive, stable application branch.
* **Context Retention:** Utilizes localized state files to meticulously track progress, preventing the system from suffering context rot or losing the "big picture" during long-running, multi-day implementation tasks.

## Relationship to the BMAD Method
While the BMAD framework defines the strict rules, personas, and architectural boundaries of our system, Loki Mode is the engine that drives those personas forward. It operationalizes Agile rituals, ensuring that the specialized BMAD agents (like the Test Engineering Architect or Systems Architect) are deployed in parallel rather than sequentially, radically accelerating the development timeline without sacrificing code quality or regulatory compliance.

## Operational Mandate
When invoked, Loki Mode prioritizes deterministic execution and strict task ordering. It ensures no feature is coded before the corresponding test cases are mapped, and no PR is merged without verifying system stability.
